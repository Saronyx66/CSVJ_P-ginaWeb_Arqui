2.1 Organización del procesador

La Unidad de Procesamiento (CPU) controla el funcionamiento del computador y lleva a cabo sus funciones de procesamiento de datos. Frecuentemente se le llama procesador. Un procesador, incluye tanto registros visibles por el usuario como registros de control/estado. Los registros visibles por el usuario pueden ser de uso general o tener una utilidad especial, mientras que los registros de control y estado se usan para controlar el funcionamiento del procesador, un claro ejemplo es el contador de programa. Lleva a cabo una gran variedad de Cálculos Comparaciones numéricas Transferencias de datos como respuesta a las peticiones de los programas que están siendo ejecutados en memoria.

2.2 Estructura de registros

2.2.1 Registros visibles para el usuaio

Los registros visibles para el usuario son aquellos que pueden ser accedidos directamente por los programadores para manipular datos y direcciones. Tipos:

  • Registros de datos:

    Almacenan valores numéricos para operaciones

  • Registro de direcciones:

    Contienen direcciones de memoria

  • Registros de propósito general:

    Usados para diferentes tareas según la necesidad del programa

  • Registros de condición:

    Guardan información sobre el resultado de una operación

2.2.2 Registros de control y de estados

Estos registros son utilizados por la CPU para gestionar su operación y supervisar el estado del sistema. Son esenciales para el control interno del procesador, ya que almacenan información sobre la ejecución de instrucciones, posibles errores y condiciones específicas del hardware. Gracias a estos registros, la CPU puede tomar decisiones sobre su funcionamiento y garantizar que los procesos se realicen de manera estable y segura.

2.2.3 Ejemplos de registros de CPU reales

  • Registros de propósito general:

    para almcanenar datos temporales y variables en operaciones

  • Registros de propósito específico:

    Incluyen el contador de programa que guarda la dirección de la siguiente instrucción y el registro de instrucción que almacena la instrucción en ejecución

  • Registros de estado y control:

    como el registro de banderas que almacena información sobre el estado del procesador

  • Registros de punto flotante:

    usados para cálculos matemáticos avanzados

  1. Arquitectura RISC-V

  2. Usa 32 registros de propósito general donde x0 siempre es 0
  3. Tiene registros especiales como el PC y el CSR
  4. Es ampliamente usada en sistemas embedidos y computación de alto rendimiento

  5. Arquitectura PowerPC (IBM)

  6. Usa 32 registros de propósito general para operaciones enteras
  7. Tiene 32 registros de punto flotante
  8. Incluye registros especiales como el LR para llamadas a subrutinas y CTR para bucles

2.3 El ciclo de instrucción

El ciclo de instrucción es el proceso mediante el cual la CPU ejecuta instrucciones, dividido en tres etapas principales:

  1. Fetch (Búsqueda):

    Se obtiene la instrucción desde la memoria y se almacena en el IR

  2. Decode (Decodificación):

    Se interpreta la instrucción y se determinan los registros y operaciones necesarias

  3. Execute (Ejecución):

    Se realizan las operaciones y se almacenan los resultados

2.3.1 Ciclo Fetch-Decode-Execute

El Ciclo de Instrucción (Fetch-Decode-Execute) es el proceso fundamental mediante el cual una CPU ejecuta instrucciones de un programa.

Fetch
El procesador obtiene la instrucción desde la memoria RAM.
La dirección de la instrucción se encuentra en el contador de programa (PC, Program Counter).
La instrucción se almacena en el Registro de Instrucción (IR, Instruction Register).
Decode
La CPU interpreta la instrucción para determinar qué operación realizar y qué operandos necesita.
Se activa la Unidad de Control para coordinar el proceso.
Execute
Si es una operación aritmética/lógica, se usa la ALU (Unidad Aritmética y Lógica).
Si implica memoria, se accede a los datos en RAM.
Si es una instrucción de control, el flujo del programa cambia (saltos, llamadas a funciones).
Los resultados pueden almacenarse en registros o en la memoria.

2.3.2 Segmentación de instrucciones

La segmentación de instrucciones es una técnica que permite implementar el paralelismo a nivel de instrucción en un único procesador.

Técnica pipeline

Esta compuesto por una lista de segmentos y secuenciales en donde cada segmento lleva acabo una tarea o un grupo de tareas computacionales.

2.3.3 Conjunto de instrucciones: Características y funciones

El conjunto de instrucciones es elconjunto de comandos e instrucciones que un procesador puede ejecutar. Define cómo el software interactúa con el hardware, especificando las operaciones que el procesador puede realizar, los tipos de datos soportados y el formato de las instrucciones

RISC

Instrucciones simples y rápidas
Menos modos de direccionamiento
Arquitecturas ARM, MIPS

CISC

Instrucciones más complejas y variadas
Mayor uso de memoria y modos de direccionamiento

2.4 Casos de estudio de CPU reales

En informática, los modos de direccionamiento son las diferentes maneras de especificar un operando dentro de una instrucción en lenguaje ensamblador. Un modo de direccionamiento especifica la forma de calcular la dirección de memoria efectiva de un operando mediante el uso de la información contenida en registros y/o constantes, contenida dentro de una instrucción de la máquina o en otra parte. No existe una forma generalmente aceptada de nombrar a los distintos modos de direccionamiento. En particular, los distintos autores y fabricantes de equipos pueden dar nombres diferentes para el modo de hacer frente al mismo, o los mismos nombres, a los diferentes modos de direccionamiento. Además, un modo de direccionamiento que en una determinada arquitectura se trata como un modo de direccionamiento, puede representar la funcionalidad que en otra arquitectura está cubierto por dos o más modos de direccionamiento.
